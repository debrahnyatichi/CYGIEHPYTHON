# 🐍 CYGIEHPYTHON

> A structured learning repository for Python and Django — from the fundamentals to full-stack web development.

## 📖 About

**CYGIEHPYTHON** is a hands-on learning repository designed to guide learners through Python programming and Django web framework concepts. Whether you're a beginner writing your first `Hello, World!` or levelling up to building full web applications with Django, this repo has you covered.

---

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed:

- [Python 3.8+](https://www.python.org/downloads/)
- [pip](https://pip.pypa.io/en/stable/)
- [Git](https://git-scm.com/)

### Installation

1. **Clone the repository**

   ```bash
   git clone https://github.com/debrahnyatichi/CYGIEHPYTHON.git
   cd CYGIEHPYTHON
   ```

2. **Create and activate a virtual environment**

   ```bash
   # Create
   python -m venv venv

   # Activate (Mac/Linux)
   source venv/bin/activate

   # Activate (Windows)
   venv\Scripts\activate
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

4. **Run any Python script**

   ```bash
   python python-basics/variables.py
   ```

5. **Run a Django project**

   ```bash
   cd Djando/projects/myproject
   python manage.py migrate
   python manage.py runserver
   ```

   Then open your browser at `http://127.0.0.1:8000`

## 📚 Topics Covered

### 🐍 Python

| Topic | Description |
|---|---|
| Variables & Data Types | Strings, integers, floats, booleans, lists, dicts, tuples |
| Control Flow | `if/elif/else`, `for`, `while`, `break`, `continue` |
| Functions | Defining, calling, args, kwargs, return values |
| Object-Oriented Programming | Classes, objects, inheritance, encapsulation |
| File Handling | Reading and writing files |
| Error Handling | `try/except/finally`, custom exceptions |
| Modules & Packages | Importing, creating modules, `__init__.py` |
| List Comprehensions | Concise and Pythonic patterns |

### 🌐 Django

| Topic | Description |
|---|---|
| Project Setup | `startproject`, `startapp`, project structure |
| Models | Defining models, migrations, database operations |
| Views | Function-based views, class-based views |
| Templates | Django template language, inheritance, context |
| URLs | URL routing and namespacing |
| Forms | Django forms, form validation |
| Admin Panel | Registering models, customising admin |
| Authentication | Login, logout, user registration |
